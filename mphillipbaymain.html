A Lightning Web Component I refactored for Accounts. This LWC gathers all contacts associated with an account, and lists their names next to their respective profile pictures.

This LWC uses an HTML, CSS, JS, and JS-Meta files, along with an Apex controller. The Apex controller uses a SOQL query to yield retrieve all Contacts associated with the Account. The Javascript uses the controller to give the HTML and CSS files the proper information to display.

Solution for a Breadth First Tree traversal. Though there are many ways do a Breadth First traversal of a Tree, I chose to use a while loop and a queue.

In the code I have built a local Node class, and filled out a tree with the Nodes storing int values. From the root of the tree I add the it too a queue and then 'poll' it which removes and returns the front object of the queue, while filling its children left to right and repeat.

This Apex Trigger works on an Opportunity Object and starts an approval process if the Opportunity is greater than or equal to $5000

For each Opportunity object that is updated or inserted to have an Opportunity.Amount over $5000, a Process Approval is declared, instantiated, initialized, and subsequently processed. This code requires that a Approval Process already be created -- in this example the name is 'X5000Approval'.

Solutions for Depth First Tree Traversals. In this code I traversed a tree depth first in Pre-Order, In-Order, and Post-Order.

This code creates its on Node class and uses it to create a tree. It then has local methods that traverse the tree in whichever order. Pre-Order is Print-RecurseLeft-Recurse-Right, In-Order is RecurseLeft-Print-RecurseRight, and Post-Order is RecurseLeft-RecurseRight-Print

A Website Scraper in Python that scrapes Recreation.gov to see if there are any available campsites for a given day, and the subsequent week out. If there is, it emails you so you can get on immediately and book it!

This is the first website scraper I wrote in Python and so it's pretty messy -- but it was at one point functional! With this code, a chron job, and a little refactoring it could be set up again relatively easily!
